# Возможные следующие шаги

Список идей для развития проекта. Порядок не строгий — приоритеты определяются по необходимости.

## 1. Выбор альтернативных рецептов

Toggle разрешённых альтов на уровне системы (или глобально). Сейчас solver всегда берёт базовый non-alternate рецепт; пользователь должен иметь возможность включать конкретные альты, чтобы посмотреть, как меняется потребление сырья и количество зданий.

- Хранить per-system список включённых альтов (например, `Set<recipeClassName>`)
- В `pickDefaultRecipe` учитывать, разрешён ли альт
- UI: чекбоксы рядом с каждой альтернативкой в детали предмета или отдельная вкладка «Рецепты» в системе

## 2. Визуальный граф цепочки (React Flow)

Сейчас результат — линейный список шагов. Граф наглядно покажет, как материалы перетекают между этапами.

- React Flow с автолэйаутом (dagre или elk)
- Узлы — здания (с количеством и тактом), рёбра — потоки предметов с rate/мин
- Группировка по типу здания или цвет по форме (solid/liquid/gas)
- Возможность редактировать рецепты прямо из графа

## 3. Экстракторы и узлы добычи

Автоматический подсчёт количества Miner Mk.1/2/3 (или Oil Extractor / Water Pump / Resource Well) для добычи raw-ресурсов с учётом:

- Чистоты узла (impure / normal / pure → ×0.5 / ×1 / ×2)
- Уровня экстрактора
- Overclock (1%–250% + power shards)

UI: для каждого raw-ресурса в результате — рекомендация «N × Miner Mk.2 на Pure-узле @ 100%» или альтернативы.

## 4. Полная LP-оптимизация

Сейчас solver жадный — берёт первый default-рецепт. Полноценная оптимизация требует линейного программирования.

- `glpk.js` или `javascript-lp-solver`
- Целевая функция: минимизация руды / энергии / зданий / какой-то взвешенной комбинации
- Переменные: rate каждого включённого рецепта (в т.ч. альтов)
- Ограничения: производство ≥ потребление для каждого предмета, source-budgets ≤ available
- UI: переключатель цели оптимизации в системе

## 5. Шаринг систем

Экспорт системы в URL или JSON для отправки кому-то.

- Сериализация системы в base64-encoded JSON в query string
- Кнопка «Поделиться» → копирует ссылку в буфер
- При открытии URL с параметром — импорт системы и предложение сохранить

## 6. Прочие идеи

- **Несколько mod-pack данных** — поддержка модов (Ficsmas, рецепты сообщества) через альтернативный data-build
- **Тёмная/светлая тема** — сейчас только тёмная
- **Production goals** — сравнение с тем, что игрок реально производит (через парс save-файла? — отдельный большой проект)
- **Power planning** — какие генераторы и сколько нужно для покрытия мощности цепочки
- **Storage / belts / pipes** — рекомендация типа конвейера/трубы по rate
- **Undo/redo** для редактирования системы
- **Импорт из save-файлов / Satisfactory Calculator** — преобразование чужих планов в наши системы
